# Spin-Flip ORMAS-CI (SF-ORMAS)

## Motivation

Standard CI methods fix the number of alpha and beta electrons and enumerate
determinants within a single $M_s$ sector. This works well when the target
state is dominated by a single configuration, but many important chemical
problems — diradicals, bond breaking, and low-lying excited states — are
inherently multiconfigurational. A single-reference method cannot describe
them well because the target wavefunction involves several determinants with
comparable weight.

The spin-flip approach circumvents this by starting from a high-spin
reference that *is* well described by a single determinant. The target-state
determinants are then generated by flipping electron spins from alpha to
beta, landing in a lower $M_s$ sector where the multiconfigurational
character lives.

## The Spin-Flip Concept

The procedure has three steps:

1. Compute a high-spin ROHF reference with
   $M_s(\text{ref}) = M_s(\text{target}) + n_\text{sf}$, where
   $n_\text{sf}$ is the number of spin flips.
2. Generate determinants in the target $M_s$ sector by applying
   $\alpha \to \beta$ spin flips to the reference configuration.
3. Diagonalize the CI Hamiltonian in this determinant space.

For a single spin-flip targeting a singlet from a triplet reference:
$M_s(\text{ref}) = +1$ and $M_s(\text{target}) = 0$. The high-spin
triplet is a single determinant even though the target singlet is
multiconfigurational.

## Spin Completeness via ORMAS

A naive spin-flip CI that applies a fixed number of flips generates an
$M_s$ space that is **not** an eigenspace of $\hat{S}^2$. The resulting
states suffer from spin contamination.

SF-ORMAS achieves spin completeness by using ORMAS subspace constraints
that include all configurations needed for the full spin-complete manifold.
The subspace occupation bounds are chosen so that every determinant required
to construct proper $\hat{S}^2$ eigenstates is present in the CI expansion.

**Example: two electrons in two orbitals.** The triplet reference is
$|\alpha\alpha\rangle$. A single spin flip generates $|\alpha\beta\rangle$
and $|\beta\alpha\rangle$, which span both:

- the singlet: $(|\alpha\beta\rangle - |\beta\alpha\rangle)/\sqrt{2}$
- the triplet $M_s = 0$ component: $(|\alpha\beta\rangle + |\beta\alpha\rangle)/\sqrt{2}$

Both spin states are present in the CI expansion, so the Hamiltonian
eigenstates are proper $\hat{S}^2$ eigenfunctions with no contamination.

## Reference State Requirements

The reference must be a **restricted open-shell Hartree–Fock** (ROHF)
calculation, not UHF. ROHF provides a single set of spatial orbitals shared
by alpha and beta electrons, which is essential for consistent spin-flip
enumeration. UHF breaks spatial symmetry between the spin channels and
complicates the mapping between reference and target determinants.

In PySCF:

```python
mol.spin = 2  # 2S for the high-spin reference (e.g., triplet)
mf = pyscf.scf.ROHF(mol)
mf.kernel()
```

## SF-ORMAS Subspace Categories

Following Mato and Gordon, the active orbitals are divided into three
categories:

**SF-CAS (spin-flip active space):** Orbitals directly involved in
bond breaking or multiconfigurational character. Full CI is performed
within this space — no occupation restrictions are applied. This is where
the spin flips occur.

**Hole space:** Doubly-occupied reference orbitals from which electrons
can be excited, bounded by a maximum number of holes. Analogous to RAS1
in standard {doc}`rasci`.

**Particle space:** Virtual reference orbitals into which electrons can
be excited, bounded by a maximum number of particles. Analogous to RAS3
in standard {doc}`rasci`.

The simplest SF-ORMAS calculation uses only the SF-CAS space. Adding hole
and particle spaces captures dynamic correlation on top of the spin-flip
multiconfigurational description.

## Electron Count Relationship

The reference and target states have different $(N_\alpha, N_\beta)$ but
the same total electron count:

$$
N_\alpha^{\text{target}} = N_\alpha^{\text{ref}} - n_\text{sf}, \qquad
N_\beta^{\text{target}} = N_\beta^{\text{ref}} + n_\text{sf}
$$

PySCF's CASCI driver uses the **target** electron counts, not the reference
counts. This is an important practical detail: the `nelecas` argument to
CASCI should reflect the $M_s = 0$ (or other target) sector.

## Single vs. Double Spin-Flip

**Single SF ($n_\text{sf} = 1$):** Triplet reference $\to$ singlet target.
The most common case, suitable for diradicals and single-bond breaking.
Example: the $\pi \to \pi^*$ twisted ethylene problem.

**Double SF ($n_\text{sf} = 2$):** Quintet reference $\to$ singlet target.
Required for systems with four strongly correlated electrons, such as
trimethylenemethane or simultaneous breaking of two bonds.

Higher spin-flip orders are possible but rarely needed in practice.

## Worked Example: Ethylene Torsion

Twisting ethylene by 90° breaks the $\pi$ bond and creates a diradical
singlet that cannot be described by a single determinant. SF-ORMAS handles
this with a CAS(2,2) single spin-flip:

1. **Reference:** ROHF triplet ($M_s = +1$) with the two $\pi/\pi^*$
   orbitals singly occupied: $|\pi^\alpha \, \pi^{*\alpha}\rangle$.
2. **Active space:** Two orbitals ($\pi, \pi^*$), two electrons. A single
   subspace with min=0, max=4 (full CI in the space).
3. **Target sector:** $M_s = 0$, giving $N_\alpha = 1$, $N_\beta = 1$.
4. **Determinants:** $|\pi^\alpha \, \pi^{*\beta}\rangle$ and
   $|\pi^\beta \, \pi^{*\alpha}\rangle$.
5. **Result:** The CI eigenstates are the spin-complete singlet and
   triplet $M_s = 0$ states.

## Relationship to Other Methods

**SF-CIS / SF-TDDFT:** Limited to single excitations from the high-spin
reference and not spin-complete. SF-ORMAS includes all configurations
within the subspace bounds, ensuring spin completeness.

**Standard CASCI:** Enumerates determinants within a fixed $M_s$ sector
without reference to a high-spin starting point. For open-shell singlets
this requires a multiconfigurational reference or careful orbital
selection. SF-ORMAS sidesteps this by using the well-defined high-spin
ROHF orbitals.

**ORMAS-CI:** SF-ORMAS is a specific application of the {doc}`ormas`
framework. It uses the same enumeration machinery and subspace constraints
but generates determinants in a different $M_s$ sector than the reference.

## Implementation in This Package

SF-ORMAS is implemented through the `SFORMASFCISolver` class and the
`SFORMASConfig` / `SpinFlipConfig` configuration objects. The solver
wraps the standard `ORMASFCISolver`, handling the mapping between
reference and target electron counts automatically. See the API
documentation for usage details.
